# Node.js Network & Protocol Level – Complete Guide

Node.js excels at network programming due to its event‑driven, non‑blocking I/O model. Understanding the protocol stack—from raw TCP sockets to high‑level message queues—is essential for building scalable, performant, and secure distributed systems. This guide covers the foundational network layers and the most important application protocols used in enterprise Node.js applications.

---

## 1. TCP Sockets

TCP (Transmission Control Protocol) provides a **reliable, ordered, stream‑oriented** connection between two endpoints. Node.js exposes TCP through the `net` module.

### TCP Server

```javascript
const net = require('node:net');

const server = net.createServer((socket) => {
  console.log('Client connected:', socket.remoteAddress, socket.remotePort);

  socket.on('data', (data) => {
    console.log('Received:', data.toString());
    socket.write(`Echo: ${data}`);
  });

  socket.on('end', () => {
    console.log('Client disconnected');
  });

  socket.on('error', (err) => {
    console.error('Socket error:', err.message);
  });
});

server.listen(3000, '0.0.0.0', () => {
  console.log('TCP server listening on port 3000');
});

// Server options
server.maxConnections = 10000;
server.on('connection', (socket) => {
  server.getConnections((err, count) => {
    console.log(`Active connections: ${count}`);
  });
});
```

### TCP Client

```javascript
const net = require('node:net');

const client = net.createConnection({ port: 3000, host: 'localhost' }, () => {
  console.log('Connected to server');
  client.write('Hello from client!');
});

client.on('data', (data) => {
  console.log('Server says:', data.toString());
  client.end();
});

client.on('end', () => {
  console.log('Disconnected from server');
});

client.on('error', (err) => {
  console.error('Connection error:', err.message);
});

// Set timeout
client.setTimeout(5000, () => {
  console.error('Connection timeout');
  client.destroy();
});
```

### Handling Backpressure in TCP Sockets

TCP sockets are streams. Writing too fast can overwhelm the kernel buffer. Always respect the return value of `socket.write()`.

```javascript
function sendLargeData(socket, data) {
  let i = 0;

  function write() {
    let ok = true;
    while (i < data.length && ok) {
      ok = socket.write(data[i]);
      i++;
    }

    if (i < data.length) {
      socket.once('drain', write);
    } else {
      socket.end();
    }
  }

  write();
}
```

### TCP Keep‑Alive

Prevent idle connections from being dropped by firewalls or detecting dead peers.

```javascript
socket.setKeepAlive(true, 60000); // Enable, send probe after 60s idle
```

### Real‑World: Custom Protocol Over TCP

```javascript
class MessageProtocol {
  static encode(message) {
    const payload = Buffer.from(JSON.stringify(message), 'utf8');
    const header = Buffer.alloc(4);
    header.writeUInt32BE(payload.length, 0);
    return Buffer.concat([header, payload]);
  }

  static decode(buffer) {
    if (buffer.length < 4) return null;
    const length = buffer.readUInt32BE(0);
    if (buffer.length < 4 + length) return null;
    const payload = buffer.subarray(4, 4 + length);
    return {
      message: JSON.parse(payload.toString('utf8')),
      remaining: buffer.subarray(4 + length)
    };
  }
}

// Usage in socket handler
let buffer = Buffer.alloc(0);
socket.on('data', (chunk) => {
  buffer = Buffer.concat([buffer, chunk]);
  let result;
  while ((result = MessageProtocol.decode(buffer)) !== null) {
    console.log('Decoded message:', result.message);
    buffer = result.remaining;
  }
});
```

## 2. HTTP/2

HTTP/2 improves upon HTTP/1.1 with **multiplexing**, **header compression**, **server push**, and **stream prioritization**.

### Key Differences from HTTP/1.1

| Feature | HTTP/1.1 | HTTP/2 |
|---------|----------|--------|
| **Multiplexing** | One request per connection (or limited pipelining) | Multiple streams over single connection |
| **Header Compression** | None | HPACK |
| **Server Push** | No | Yes |
| **Binary Protocol** | Text‑based | Binary framing |
| **Stream Prioritization** | No | Yes |

### HTTP/2 Server with Node.js

```javascript
const http2 = require('node:http2');
const fs = require('node:fs');

const server = http2.createSecureServer({
  key: fs.readFileSync('server.key'),
  cert: fs.readFileSync('server.crt')
});

server.on('stream', (stream, headers) => {
  const path = headers[':path'];
  const method = headers[':method'];

  console.log(`${method} ${path}`);

  if (path === '/') {
    stream.respond({
      'content-type': 'text/html',
      ':status': 200
    });
    stream.end('<h1>HTTP/2 Server</h1>');
  } else if (path === '/api/data') {
    // Server Push example
    stream.pushStream({ ':path': '/api/styles' }, (err, pushStream) => {
      if (!err) {
        pushStream.respond({ ':status': 200, 'content-type': 'text/css' });
        pushStream.end('body { font-family: sans-serif; }');
      }
    });

    stream.respond({
      'content-type': 'application/json',
      ':status': 200
    });
    stream.end(JSON.stringify({ message: 'Hello HTTP/2' }));
  } else {
    stream.respond({ ':status': 404 });
    stream.end('Not Found');
  }
});

server.listen(3443, () => {
  console.log('HTTP/2 server on https://localhost:3443');
});
```

### HTTP/2 Client

```javascript
const http2 = require('node:http2');

const client = http2.connect('https://localhost:3443', {
  ca: fs.readFileSync('server.crt')
});

client.on('error', (err) => console.error('Client error:', err));

const req = client.request({ ':path': '/api/data' });

req.on('response', (headers) => {
  console.log('Response headers:', headers);
});

let data = '';
req.on('data', (chunk) => (data += chunk));
req.on('end', () => {
  console.log('Response:', data);
  client.close();
});

req.end();
```

### Server Push

Server push allows the server to send resources proactively before the client requests them.

```javascript
stream.on('push', (pushStream, headers) => {
  console.log('Server pushed:', headers[':path']);
});
```

### Multiplexing in Action

With HTTP/2, multiple requests can be in flight simultaneously over a single TCP connection, eliminating head‑of‑line blocking.

## 3. TLS Encryption (SSL/TLS)

TLS (Transport Layer Security) encrypts data in transit. Node.js uses OpenSSL under the hood via the `tls` module.

### Creating TLS Server and Client

```javascript
const tls = require('node:tls');
const fs = require('node:fs');

// TLS Server
const server = tls.createServer({
  key: fs.readFileSync('server-key.pem'),
  cert: fs.readFileSync('server-cert.pem'),
  ca: fs.readFileSync('client-cert.pem'), // For mutual TLS
  requestCert: true,
  rejectUnauthorized: true
}, (socket) => {
  console.log('Secure connection established');
  console.log('Client certificate:', socket.getPeerCertificate());

  socket.write('Welcome to TLS server!\n');
  socket.pipe(socket); // Echo
});

server.listen(8000, () => {
  console.log('TLS server on port 8000');
});

// TLS Client
const client = tls.connect(8000, 'localhost', {
  key: fs.readFileSync('client-key.pem'),
  cert: fs.readFileSync('client-cert.pem'),
  ca: fs.readFileSync('server-cert.pem'),
  rejectUnauthorized: true
}, () => {
  console.log('Connected to server');
  console.log('Server certificate:', client.getPeerCertificate());
  client.write('Hello from client!');
});

client.on('data', (data) => console.log('Received:', data.toString()));
client.on('end', () => console.log('Connection closed'));
```

### Generating Certificates (Development)

```bash
# Self‑signed certificate
openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes

# For mutual TLS, generate client certificates similarly
```

### HTTPS Server with TLS

```javascript
const https = require('node:https');
const fs = require('node:fs');

const options = {
  key: fs.readFileSync('key.pem'),
  cert: fs.readFileSync('cert.pem'),
  // Modern TLS configuration
  minVersion: 'TLSv1.2',
  ciphers: 'HIGH:!aNULL:!MD5'
};

https.createServer(options, (req, res) => {
  res.writeHead(200);
  res.end('Secure HTTPS');
}).listen(443);
```

### Mutual TLS (mTLS)

mTLS authenticates both client and server using certificates. Essential for service‑to‑service communication in zero‑trust environments.

```javascript
const server = https.createServer({
  key: fs.readFileSync('server-key.pem'),
  cert: fs.readFileSync('server-cert.pem'),
  ca: fs.readFileSync('client-ca.pem'), // Trusted client CAs
  requestCert: true,
  rejectUnauthorized: true // Reject if client certificate invalid
}, (req, res) => {
  const clientCert = req.socket.getPeerCertificate();
  console.log('Authenticated client:', clientCert.subject.CN);
  res.end('mTLS authenticated');
});
```

## 4. DNS

DNS (Domain Name System) resolution translates domain names to IP addresses. Node.js provides both **blocking** (`dns.lookup`) and **non‑blocking** (`dns.resolve*`) methods.

### `dns.lookup` vs `dns.resolve`

| Method | Uses | Thread Pool | Blocking? |
|--------|------|-------------|-----------|
| `dns.lookup` | OS resolver (`getaddrinfo`) | Yes | Uses libuv thread pool |
| `dns.resolve` | Direct DNS queries via c‑ares | No | True async |

**Always prefer `dns.resolve*` in production** to avoid thread pool exhaustion.

```javascript
const dns = require('node:dns');

// ❌ Blocking thread pool (but convenient)
dns.lookup('example.com', (err, address, family) => {
  console.log('lookup:', address, family);
});

// ✅ Non‑blocking
dns.resolve4('example.com', (err, addresses) => {
  console.log('resolve4:', addresses);
});

dns.resolveMx('gmail.com', (err, exchanges) => {
  console.log('MX records:', exchanges);
});

dns.resolveTxt('example.com', (err, records) => {
  console.log('TXT records:', records);
});
```

### Custom DNS Servers and Caching

```javascript
const dns = require('node:dns');

// Set custom DNS servers (e.g., Cloudflare, Google)
dns.setServers(['1.1.1.1', '8.8.8.8']);

// Implement DNS caching
const dnsCache = new Map();
const CACHE_TTL = 300000; // 5 minutes

async function resolveWithCache(hostname) {
  const cached = dnsCache.get(hostname);
  if (cached && Date.now() < cached.expires) {
    return cached.addresses;
  }

  return new Promise((resolve, reject) => {
    dns.resolve4(hostname, (err, addresses) => {
      if (err) return reject(err);
      dnsCache.set(hostname, {
        addresses,
        expires: Date.now() + CACHE_TTL
      });
      resolve(addresses);
    });
  });
}
```

### Reverse DNS

```javascript

  console.log('Reverse lookup:', hostnames);
});
```

## 5. WebSockets

WebSockets provide **full‑duplex, low‑latency** communication over a single TCP connection. Ideal for real‑time applications.

### Basic WebSocket Server with `ws`

```javascript
const WebSocket = require('ws');

const wss = new WebSocket.Server({ port: 8080 });

wss.on('connection', (ws, req) => {
  const ip = req.socket.remoteAddress;
  console.log(`Client connected from ${ip}`);

  ws.on('message', (data, isBinary) => {
    console.log('Received:', isBinary ? 'binary' : data.toString());
    // Echo back
    ws.send(`Echo: ${data}`);
  });

  ws.on('close', () => console.log('Client disconnected'));
  ws.on('error', console.error);

  // Send welcome message
  ws.send('Connected to WebSocket server');
});
```

### WebSocket Client (Browser)

```javascript
const ws = new WebSocket('ws://localhost:8080');

ws.onopen = () => {
  console.log('Connected');
  ws.send('Hello Server!');
};

ws.onmessage = (event) => {
  console.log('Received:', event.data);
};

ws.onclose = () => console.log('Disconnected');
```

### Scaling WebSockets with Redis Pub/Sub

When running multiple Node.js instances, WebSocket connections must be synchronized.

```javascript
const WebSocket = require('ws');
const Redis = require('ioredis');

const redis = new Redis();
const wss = new WebSocket.Server({ port: 8080 });

// Subscribe to Redis channel
redis.subscribe('broadcast', (err) => {
  if (err) console.error('Redis subscription error:', err);
});

redis.on('message', (channel, message) => {
  // Broadcast to all connected clients
  wss.clients.forEach((client) => {
    if (client.readyState === WebSocket.OPEN) {
      client.send(message);
    }
  });
});

wss.on('connection', (ws) => {
  ws.on('message', (data) => {
    // Publish to Redis for other instances
    redis.publish('broadcast', data.toString());
  });
});
```

### Socket.IO — Higher‑Level Abstraction

```javascript
const { Server } = require('socket.io');
const io = new Server(3000, {
  cors: { origin: '*' }
});

io.on('connection', (socket) => {
  console.log('User connected:', socket.id);

  socket.on('chat message', (msg) => {
    io.emit('chat message', msg); // Broadcast to all
  });

  socket.on('disconnect', () => {
    console.log('User disconnected');

```

### WebSocket Authentication


## 6. gRPC

gRPC is a **high‑performance RPC framework** using Protocol Buffers (protobuf) for serialization and HTTP/2 for transport.

### Why gRPC?

- **Binary serialization** (faster than JSON)
- **Strongly typed contracts** via `.proto` files
- **Bidirectional streaming**
- **Built‑in deadlines/cancellation**
- **Polyglot interoperability**

### Defining a Service (`.proto`)

```protobuf
syntax = "proto3";

package user;

service UserService {
  rpc GetUser (GetUserRequest) returns (User) {}
  rpc ListUsers (ListUsersRequest) returns (stream User) {}
  rpc CreateUser (CreateUserRequest) returns (User) {}
  rpc Chat (stream ChatMessage) returns (stream ChatMessage) {}
}

message GetUserRequest {
  string id = 1;
}

message User {
  string id = 1;
  string name = 2;
  string email = 3;
}

message ListUsersRequest {
  int32 page = 1;
  int32 limit = 2;
}

message CreateUserRequest {
  string name = 1;
  string email = 2;
}

message ChatMessage {
  string user_id = 1;
  string text = 2;
}
```

### gRPC Server with Node.js

```javascript
const grpc = require('@grpc/grpc-js');
const protoLoader = require('@grpc/proto-loader');

const packageDefinition = protoLoader.loadSync('user.proto', {
  keepCase: true,
  longs: String,
  enums: String,
  defaults: true,
  oneofs: true
});

const userProto = grpc.loadPackageDefinition(packageDefinition).user;

const users = new Map();

const server = new grpc.Server();

server.addService(userProto.UserService.service, {
  getUser: (call, callback) => {
    const user = users.get(call.request.id);
    if (!user) {
      return callback({ code: grpc.status.NOT_FOUND, message: 'User not found' });
    }
    callback(null, user);
  },

  listUsers: (call) => {
    const { page, limit } = call.request;
    const userList = Array.from(users.values());
    for (const user of userList.slice((page - 1) * limit, page * limit)) {
      call.write(user);
    }
    call.end();
  },

  createUser: (call, callback) => {
    const id = Date.now().toString();
    const user = { id, ...call.request };
    users.set(id, user);
    callback(null, user);
  },

  chat: (call) => {
    call.on('data', (message) => {
      console.log(`[${message.user_id}]: ${message.text}`);
      call.write({
        user_id: 'server',
        text: `Received: ${message.text}`
      });
    });
    call.on('end', () => call.end());
  }
});

server.bindAsync('0.0.0.0:50051', grpc.ServerCredentials.createInsecure(), () => {
  console.log('gRPC server running on port 50051');
});
```

### gRPC Client

```javascript

const packageDefinition = protoLoader.loadSync('user.proto', {});

chat.write({ user_id: 'client1', text: 'Hello!' });
chat.on('data', (msg) => console.log('Server says:', msg.text));
```

### gRPC Interceptors

```javascript

function loggingInterceptor(options, nextCall) {
  return new grpc.InterceptingCall(nextCall(options), {
    start: (metadata, listener, next) => {
      console.log('Request:', metadata.get('grpc-timeout'));
      next(metadata, {

          next(metadata);

          console.log('Response message:', message);
          next(message);
        },
        onReceiveStatus: (status, next) => {
          console.log('Status:', status);
          next(status);
        }
      });
    }
  });
}

  'localhost:50051',
  credentials,
  { interceptors: [loggingInterceptor] }
```

## 7. Message Queues (Kafka / RabbitMQ / NATS)

Message queues decouple producers and consumers, enabling **asynchronous processing**, **load leveling**, and **fault tolerance**.

### Comparison

| Feature | Kafka | RabbitMQ | NATS |
|---------|-------|----------|------|
| **Primary Use** | Event streaming, high throughput | Reliable messaging, complex routing | Ultra‑low latency, simple pub/sub |
| **Protocol** | Custom binary | AMQP 0‑9‑1 | Simple text/binary |
| **Message Ordering** | Per partition | FIFO per queue | No (JetStream adds ordering) |
| **Persistence** | Yes (log‑based) | Yes (queues) | Optional (JetStream) |
| **Throughput** | Very high (>1M msg/s) | Moderate (~50K msg/s) | Very high (>10M msg/s) |
| **Latency** | Milliseconds | Milliseconds | Microseconds |
| **Complexity** | High (ZooKeeper/KRaft) | Medium | Low |

### RabbitMQ with `amqplib`

**Producer:**
```javascript
const amqp = require('amqplib');

async function produce() {
  const connection = await amqp.connect('amqp://localhost');
  const channel = await connection.createChannel();

  const exchange = 'orders';
  await channel.assertExchange(exchange, 'topic', { durable: true });

  const message = JSON.stringify({ orderId: '123', amount: 99.99 });
  channel.publish(exchange, 'order.created', Buffer.from(message), {
    persistent: true,
    headers: { 'x-retry-count': 0 }
  });

  console.log('Message published');
  setTimeout(() => connection.close(), 500);
}
```

**Consumer:**
```javascript
async function consume() {
  const connection = await amqp.connect('amqp://localhost');
  const channel = await connection.createChannel();

  const exchange = 'orders';
  await channel.assertExchange(exchange, 'topic', { durable: true });

  const q = await channel.assertQueue('order-processor', { durable: true });
  await channel.bindQueue(q.queue, exchange, 'order.*');

  channel.prefetch(1); // Fair dispatch

  channel.consume(q.queue, async (msg) => {
    const content = JSON.parse(msg.content.toString());
    console.log('Processing:', content);

    try {
      await processOrder(content);
      channel.ack(msg);
    } catch (err) {
      // Retry with exponential backoff
      const retryCount = (msg.properties.headers['x-retry-count'] || 0) + 1;
      if (retryCount < 3) {
        channel.publish(exchange, msg.fields.routingKey, msg.content, {
          persistent: true,
          headers: { 'x-retry-count': retryCount }
        });
      } else {
        channel.publish(exchange, 'order.failed', msg.content, { persistent: true });
      }
      channel.ack(msg); // Remove from queue after handling
    }
  });
}
```

**Consumer (with Consumer Group):**
```javascript
const consumer = kafka.consumer({ groupId: 'user-event-processor' });

async function consume() {
  await consumer.connect();
  await consumer.subscribe({ topic: 'user-events', fromBeginning: true });

  await consumer.run({
    eachMessage: async ({ topic, partition, message }) => {
      const value = JSON.parse(message.value.toString());
      console.log(`Received: ${value.event} from ${message.key}`);

      // Commit offset manually for at‑least‑once semantics
      await consumer.commitOffsets([{
        topic, partition,
        offset: (Number(message.offset) + 1).toString()
      }]);
    }
  });
}
```

### NATS with `nats`

```javascript
const { connect } = require('nats');

async function run() {
  const nc = await connect({ servers: ['nats://localhost:4222'] });

  // Simple pub/sub
  const sub = nc.subscribe('notifications');
  (async () => {
    for await (const msg of sub) {
      console.log('Received:', msg.string());
    }
  })();

  nc.publish('notifications', 'Hello NATS!');

  // Request‑Reply
  const reply = await nc.request('service.ping', 'ping', { timeout: 1000 });
  console.log('Reply:', reply.string());

  // JetStream (persistent)
  const js = nc.jetstream();
  await js.publish('events.orders', JSON.stringify({ orderId: 456 }));

  const jsSub = await js.subscribe('events.orders');
  for await (const m of jsSub) {
    console.log('JetStream:', m.string());
    m.ack();
  }
}
```

### Message Queue Patterns

**Competing Consumers (Work Queue):**
Multiple consumers on same queue; each message processed once.

**Publish/Subscribe:**
Producer sends to exchange/topic; multiple consumers each get copy.

**Request/Reply:**
Client sends request, waits for single response (NATS, RabbitMQ RPC).

**Dead Letter Queue (DLQ):**
Failed messages moved to DLQ for later inspection.

```javascript
// RabbitMQ DLQ setup
await channel.assertExchange('dlx', 'topic', { durable: true });
await channel.assertQueue('order-dlq', { durable: true });
await channel.bindQueue('order-dlq', 'dlx', '#');

await channel.assertQueue('order-processor', {
  durable: true,
  arguments: {
    'x-dead-letter-exchange': 'dlx',
    'x-dead-letter-routing-key': 'order.failed',
    'x-message-ttl': 30000 // 30 seconds
  }
});
```

## Summary Table

| Technology | Best For | Node.js Library |
|------------|----------|-----------------|
| **TCP Sockets** | Custom protocols, low‑level control | `net` |
| **HTTP/2** | Modern web servers, multiplexing | `http2` |
| **TLS** | Encryption everywhere | `tls`, `https` |
| **DNS** | Service discovery, caching | `dns` |
| **WebSockets** | Real‑time bidirectional | `ws`, `socket.io` |
| **gRPC** | Microservices, streaming | `@grpc/grpc-js` |
| **RabbitMQ** | Complex routing, reliability | `amqplib` |
| **Kafka** | Event streaming, high throughput | `kafkajs` |
| **NATS** | Ultra‑low latency, simplicity | `nats` |

**Key Takeaways:**
1. **Always use TLS in production** — even for internal services
2. **Prefer `dns.resolve*` over `dns.lookup`** to avoid thread pool starvation
3. **Use HTTP/2 for modern web APIs** — multiplexing reduces latency
4. **WebSockets need a pub/sub backend** (like Redis) for horizontal scaling
5. **gRPC + Protobuf** is excellent for service‑to‑service communication
6. **Choose message queues based on requirements** — Kafka for events, RabbitMQ for reliability, NATS for speed
